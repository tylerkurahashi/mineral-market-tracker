# セッションログ記録

## 手順

1. 以下のコマンドで東京時刻（JST）のファイル名を生成する：
   ```bash
   TZ='Asia/Tokyo' date '+%Y-%m-%d_%H-%M'
   ```
   → 保存先: `/workspace/session_log/<timestamp>.md`

2. `/workspace/session_log/` ディレクトリの存在を確認する（なければ作成）

3. このセッションの**会話全体を最初から最後まで時系列で**振り返り、ログを作成する

4. Write ツールでファイルを書き出す

---

## 記録の原則

### 基本方針

- ユーザーの発言とClaudeの応答を**時系列・会話の流れに沿って**漏れなく記録する
- ユーザーの意図・質問・指示は原文に忠実に記載する
- Claudeが実施した調査・分析・判断の内容を具体的に記録する
- コードの変更は**変更前後を両方**記載する
- コマンドの実行結果・エラーメッセージ・数値は省略しない
- 「なぜその判断をしたか」の根拠も記録する

### 記録すべき内容（漏らさないこと）

- ユーザーの最初の依頼と、その後の追加指示・方針変更
- Claudeが行った調査・探索とその結果
- エラーの原因特定プロセス（仮説→検証→結論）
- 実施した修正・変更の詳細（ファイルパス、変更前後のコード）
- 実行したコマンドと出力結果
- 数値結果（精度、エラーコード等）
- 未解決の課題・今後の引き継ぎ事項
- ユーザーからのフィードバックとそれへの対応

```markdown

**日時（JST）:** YYYY-MM-DD HH:MM
**概要:** <このセッションで何をしたかを1〜2文で>

---

```

```
