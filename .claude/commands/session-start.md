# セッション開始ブリーフィング

## 手順

### Step 1: セッションログの一覧取得

```bash
ls -1 /workspace/session_log/ | sort -r
```

直近 $ARGUMENTS 件を読み込む。`$ARGUMENTS` が空の場合は **3件**。

### Step 2: ログの読み込み

Step 1 で得たファイルを新しい順に Read ツールで読み込む。
複数ファイルがある場合は**並列**で読み込む。

### Step 3: 内容の整理・要約

読み込んだログを以下の観点で整理する:

- **情報が矛盾する場合は最新ログを優先する**
- 実施済み作業は簡潔に、未解決タスクは漏れなく抽出する

### Step 4: ブリーフィングの出力

以下のフォーマットでユーザーに提示する:

```markdown
## プロジェクト現状ブリーフィング

### 直近の作業（最終セッション: YYYY-MM-DD）
- <実施内容を箇条書きで>

### 現在の状態
- **データ:** <何が揃っているか>
- **コード:** <何が動くか・何が実装済みか>

### 未解決タスク（次にやること）
1. <優先度高>
2. <優先度中>
3. ...

### 注意事項
- <CRS変換の必要性、既知のバグ、制約など>
```

### Step 5: 問いかけ

ブリーフィング提示後、以下を問いかける:

> 何から着手しますか？
