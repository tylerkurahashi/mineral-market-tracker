# 鉱物相場トラッカー 実装計画

## 目的

ルース・鉱物標本の相場を毎日自動収集・記録し、種類別の価格推移をレポート化。
相場より割安な「買うべき商品」を自動ピックアップする。

## アーキテクチャ

```
[eBay Browse API] ─┐
[ヤフオク検索]      ├→ 収集 → 正規化(サイズ/重量/通貨) → Parquet保存
[ミネラルショップ] ─┘            ↓
                        品質評価 (Claude vision, 任意)
                            ↓
                  相場統計 → 割安検出 → 日次HTMLレポート
```

## データフロー(日次バッチ)

1. **collect**: 各ソースから対象鉱物の出品を検索・取得
2. **normalize**: タイトル/説明からサイズ(mm)・重量(g/ct)を抽出、通貨をJPYに統一
3. **store**: `data/listings/date=YYYY-MM-DD/*.parquet` にパーティション保存
4. **quality** (任意): 画像をClaude APIで評価(透明度・色・傷・総合 0-10)
5. **analyze**: 種別ごとの単価統計(g単価/中央値/分布)を計算、割安判定
6. **report**: HTMLレポート + 価格推移グラフを `reports/` に出力

## データ管理

- 形式: **Parquet**(日付パーティション)。集計は **DuckDB** でSQL照会
- 画像: URLのみ保存(品質評価時にオンデマンド取得)。必要ならローカルキャッシュ
- スキーマ: `src/mineral_tracker/models.py` の `Listing` 参照

## 割安検出ロジック

1. 種別×直近30日の「グラム単価」「サイズ単価」中央値を基準相場とする
2. 基準相場の一定割合(既定60%)未満の現役出品を候補化
3. 品質スコアがしきい値以上のものを最終ピックアップ
4. レポート冒頭に「本日の買い候補」として掲載

## データソースと注意点

| ソース | 方式 | 備考 |
|---|---|---|
| eBay | Browse API(公式) | 要開発者登録・APIキー。規約上安全 |
| ヤフオク | 検索ページ取得 | 公式APIなし。低頻度・低負荷で運用。**規約変更リスクあり、自己責任** |
| ミネラルショップ | サイト別スクレイパー | robots.txt尊重。設定でON/OFF |

## 実行環境

- GitHub Actions で毎日1回実行(cron)。結果のParquet/レポートをリポジトリにcommit
- ローカルでも `python scripts/run_daily.py` で同一処理を実行可能

## 段階的リリース

- **Phase 1** (本実装): 収集→保存→統計→レポート→割安検出の骨格一式
- **Phase 2**: ヤフオク落札(SOLD)価格の取り込み、ショップ追加、為替の精緻化
- **Phase 3**: 品質評価の常時運用、独自の品質基準チューニング、通知(Slack/メール)

## コスト

- eBay API: 無料枠内(5,000call/日)
- Claude vision: 1画像 ≒ $0.001–0.01(Haiku)。既定はOFF、`--quality` フラグで有効化
- GitHub Actions: publicリポジトリなら無料
