# 001 カテゴリ収集 + 多層分類

## 背景 / 課題
石種をキーワードで事前列挙する現方式は、鉱物種が膨大で変種名・表記揺らぎも多いため
列挙しきれない。追加要望が続くたびに `stones.yaml` を手で足す運用は破綻する。

- 例: 変種名（スペクトロライト=ラブラドライト、リチア電気石=トルマリン）
- 例: 表記揺らぎ（ビリオム石 / ビリオーム石 / ヴィリオーマイト、カイヤナイト / カヤナイト）

## 決定（2026-07-10）
**「収集」と「分類」を分離する。**

### 収集（列挙不要）
鉱物カテゴリを丸ごとページング取得する。
- ヤフオク: カテゴリ **20776（原石）** を `search?auccat=20776&b=<offset>` で巡回（実証済み）
- Yahoo!フリマ: カテゴリ / 広域キーワードで取得
- 既存のキーワード検索は「希少種の取りこぼし防止」に併用可（任意）

### 分類（表記揺らぎ対応）— 上から順に解決し、安い層で片付ける
| 層 | 手法 | 吸収する揺らぎ | 状態 |
|----|------|--------------|------|
| ① 正規化 | NFKC＋長音符ー除去＋ヴ→ビ統一＋小書き統一 | 軽微な表記揺れ | ← 今回実装 |
| ② エイリアス辞書 | `stones.yaml` の `aliases` に既知シノニム | 既知の別名 | ← 今回実装 |
| ③ LLM フォールバック | 未分類タイトルのみ Claude で正規種名へ。新種も発見 | 別転写・新種・誤記 | 保留（①②で不足なら着手） |
| ④ その他 | 未分類の頻出語をレポート表示→②へ手動昇格 | 発見キュー | ← 今回実装 |

`stones.yaml` は「収集キーワード表」から「育っていく**分類辞書**」へ役割変更。

## 今回のスコープ（①②＋カテゴリ収集）
1. `normalize.normalize_kana()` / `classify_species()`
2. `config.build_species_matcher()`（name_ja + species + aliases + 単語キーワードから別名集合を構築）
3. コレクターにカテゴリ browse を追加。取得アイテムはタイトルから種を分類（未知は `other`）
4. `sources.yaml` にカテゴリ設定、`run_daily` で matcher を配線
5. レポートに「その他（未分類）」頻出語セクション

## Yahoo!フリマの 429 について（訂正）
一時「JSボット検知で静的取得不可（要Playwright）」と判断したが誤り。実際は
**短時間に叩きすぎたことによる一時的なレート制限（ソフトBAN）**で、冷ませば plain
requests（`__NEXT_DATA__`）で復活する。curl_cffi でも弾かれたのは BAN 中はブラウザ
相当でもチャレンジCookieを要求されるため。対策は「丁寧に叩く」だけ:
- `request_interval_sec` を広め（6s〜）
- `browse_keywords` を少数に絞りリクエスト数を抑える
- 開発中の連打を避ける（`enrich_from_detail` 等の高頻度アクセスに注意）
